Data Table

This table gets populated in such a way:

  • Daily, at 17:00 (with a help of cron job) a weather entry is fetched from an api
  • It is stored to the DB and then displayed in this table
  • User submits answers for a specific day
  • A check is made if there is a row with a weather entry of the same day that the answer was submitted
  • If there is not - answers are not getting displayed
  • If thre is - answers are getting displayed only for that specific day of the weather entry
  • So in theory weather fetching is crucial - without it - the whole thing breaks :D And weather api is the thing I do not control.. so..
